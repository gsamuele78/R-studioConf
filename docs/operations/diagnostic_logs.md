# BIOME-CALC Diagnostic & Forensic Logs

This document outlines the comprehensive forensic footprint generated by the BIOME-CALC architecture. When troubleshooting crashes, unexpected behavior, or OOM errors, these logs are your primary source of truth.

## 1. Failsafe Boot Log
* **Path:** `/tmp/biome_boot_errors_<pid>.log`
* **What it is:** The earliest possible error capture mechanism.
* **When to check:** If R sessions fail to start entirely or workers die before their diagnostic logs are set up.
* **Details:** Captures failures during critical bootstrapping (e.g., CPU detection, `/Rtmp` directory scaffolding). Because the main logging infrastructure relies on successful initialization, this fallback log operates independently to ensure no error is truly "silent."

## 2. Worker Post-Mortem Logs (PSOCK Workers)
* **Path:** `/Rtmp/biome_<user>/cluster_logs/psock_<timestamp>_pid<pid>.log`
* **What it is:** Contains the raw `stdout` and `stderr` for every PSOCK worker node.
* **When to check:** If a cluster worker dies mid-calculation, resulting in errors like `unserialize(node$con) : error reading from connection` in the parent session.
* **Details:** Can be easily inspected from an active session using the `biome_worker_diagnostics()` helper. This is vital for diagnosing errors inside heavy distributed workloads (e.g., NIMBLE, Stan).

## 3. Global Diagnostic Debug Log
* **Path:** `/tmp/biome_debug_<user>_<pid>.log`
* **What it is:** A massively verbose audit trail of the entire profile initialization.
* **When to check:** When diagnosing environment setup failures, resource quota miscalculations, or deferred package hook logic.
* **Details:** This log is only generated if the `BIOME_DEBUG=1` environment variable is exported in the shell *before* starting the R session. It records every `.biome_mark_time()` milestone and any swallowed errors caught by `.biome_swerr()`.

## 4. Worker Fast-Path Trace Log
* **Path:** `/tmp/biome_worker_<pid>.log`
* **What it is:** A lightweight trace proving a worker successfully bypassed the heavy interactive profile.
* **When to check:** When verifying that PSOCK workers are correctly receiving their `BIOME_WORKER_MODE` and compiling to the local `/Rtmp` disk instead of NFS.
* **Details:** Only generated if `BIOME_WORKER_DEBUG=1` is exported in the parent session before creating the cluster.

## 5. System Audit Log
* **Path:** `/var/log/biome-log/r_biome_system.log` (Injected via the `%%LOG_FILE%%` template variable)
* **What it is:** The persistent system log capturing `sys_log()` events.
* **When to check:** For retrospective auditing of user sessions, resource limits applied, and system warnings across the entire host.

## 6. OOM (Out-Of-Memory) Marker
* **Path:** `~/ULTIMO_CRASH_RAM.txt`
* **What it is:** A special marker file placed in the user's home directory.
* **When to check:** If users complain that their session spontaneously disconnected without warning.
* **Details:** Created by external system-level monitors (such as systemd OOM watchers). If the profile detects this file on startup, it alerts the user with a `WARNING: Previous session killed by OOM` banner and automatically deletes the marker.
